## 1. 产品战略

### 1.1 产品定位

Loka 是一个 **多AI Agent 协作平台**，让用户和 AI Agent 像同事一样协作完成各种知识工作。平台有两个核心：

- **Super Agent**：一套 Multi-Agent 系统，用户用自然语言下达任务，多个专业 Agent 并行工作交付结果。能力覆盖投资分析、市场调研、竞品研究、任务管理等场景。
- **Chats（人机协作社交）**：AI Agent 作为群组一等成员参与讨论和任务执行。不是 Bot 附属品，而是有名字、有记忆、有性格的协作伙伴。
- **Cash flow** ——简化、很容易使用、降低摩擦力
- **Payment** ——找场景、参考 tempo

**总结：** Loka 是一个你和 AI Agent 一起工作的地方——做投资分析、调研竞品、推进团队任务，全在一个平台里完成。

### 1.2 解决什么问题

| 场景 | 痛点 | Loka 的解法 |
|------|------|-------------|
| 投资分析 | 1. 渠道分散，信息来源很多<br>2. 没有好的判断策略<br>3. 涉及资金，对准确度要求更高 | 1. 信息聚合，专门负责不同渠道的数据采集<br>2. 多agent共识，提供更好的交易策略<br>3. 私有数据管道，准确性比通用agent更高——待定 |
| 人与AI的协作 | 1. 工具碎片化，研究用 Perplexity，讨论用 Telegram，任务管理用 Notion，AI 用 ChatGPT<br>2. Telegram / 微信群全靠人发言，缺乏agent的参与 | 1. 分析、讨论、任务在同一个平台，Agent 贯穿始终<br>2. 群里有 AI Agent 实时参与，@一下就能出分析、做总结、推进任务 |

**应用场景：**

**投研场景：**
- "调研xx公司背景，是否值得投资"
- "调研东南亚的外卖市场"
- "我想买xx美元的xx资产，评估投资风险"

**群聊场景：**
- a和b在群聊中讨论kyb的实现，@agent 可直接给出方案和推荐的第三方服务商
- A、b、c 进行了会议并分配任务，会议结束后agent自动发布总结并@对应成员负责的任务
- a创建了预测市场agent并总结下单策略，拉入粉丝进群围观跟单

### 1.3 产品特色

**特色一：Super Agent 不只做投资**

投资分析是 Super Agent 的核心能力之一，但它同样能做竞品调研、行业分析、项目尽调、任务拆解与跟踪。用户对它说"帮我调研一下东南亚的外卖市场"或者"帮我比较一下 Figma 和 Sketch 的优劣势"，它都能调度多个专业 Agent 并行工作，交付一份结构化的研究报告。

**特色二：AI Agent 是群成员，不是 Bot**

在 Loka 的群聊中，AI Agent 有独立的名字、性格、记忆。你在 A 群里让 Agent 分析过的内容，到 B 群里它还记得。这跟 Telegram Bot 的"无状态应答"完全不同。Agent 是你的同事，不是工具。

**特色三：从分析到协作的闭环**

Super Agent 生成的分析报告可以一键分享到群聊，群里的同事和 Agent 可以接着讨论、追问、推进下一步。分析不是孤立的个人行为，而是协作的起点。

### 1.4 竞争分析

| 竞品类型 | 代表 | 目标用户 | 它们的强项 | Loka 的差异 |
|----------|------|----------|-----------|-------------|
| 通用 AI | ChatGPT, Perplexity | 所有人 | 通用问答能力强 | 多agent协作+私有数据管道，在结果的准确性上更高 |
| AI 投研工具 | Surf AI, Genspark | 专业投资者、机构 | 垂类分析深度高 | 投资数据范围更广，自带群聊社交属性 |
| 投资社交 | Stocktwits, 雪球 | 散户 | UGC 社区活跃 | 内容全靠人工，信噪比低。Loka 通过 AI Agent 参与群聊注入专业分析能力，提升社区内容质量 |
| 人机协作 IM | Teamily | 企业团队 | AI 作为群成员概念领先 | Loka 以投资/研究分析切入，在垂直场景做更深 |
| 团队协作 | Slack, Notion | 企业团队 | 工作流成熟 | AI 作为有记忆能输出的成员，而不仅是辅助工具 |

**Loka 的位置：** 在 Teamily（人机协作）和 Surf AI（投资分析）之间，做一个"能深度分析也能协作执行"的 AI 原生平台。投资分析是切入点，但平台能力不止于此。

### 1.5 市场策略 (Go-to-Market)

#### 阶段一：冷启动——解决"初期没有用户"的问题

**核心思路：** 初期不依赖用户生成内容，而是用 AI 自动生产高质量内容抢占搜索和社交流量。

**1) AI 内容工厂**
- 用 Super Agent 批量生成当下热点的投资分析报告（如"英伟达 Q4 财报解读""Polymarket 大选预测市场分析"）。
- 将这些报告发布到 Twitter、LinkedIn、Medium、dev.to 等平台，署名 Loka，文末附平台链接。
- 报告质量必须高于市面上的通用 AI 输出（有图表、有数据引用、有结论），通过内容专业度建立品牌认知。
- **目标指标：** 每周产出 5-10 篇高质量分析报告，覆盖当周投资热点。

**2) 社交媒体矩阵运营**
- 建立 Loka 官方 Twitter 账号，日常发布 Super Agent 的分析片段（适合传播的截图/金句）。
- 在投资类话题（FinTwit、#investing、#stockmarket）下用专业观点互动，不做硬广，而是用分析质量吸引关注。
- 重点关注和互动 KOL（投资博主、分析师、科技评论人），争取被引用或转发。

**3) 种子用户直接触达**
- 在 Reddit（r/investing、r/stocks）、Discord 投资社区、ProductHunt 等平台发布产品，收集第一批种子用户。
- 提供内测邀请码机制，制造稀缺感。前 500 名注册用户永久解锁 Pro 功能。
- 直接私信 Twitter 上活跃的投资爱好者，邀请试用并给反馈。

**4) Founder 侧冷启动**
- 用 TrustMR API 预填充 Discover 页面，确保上线第一天就有内容可看，不出现空货架。
- 主动邀请 10-20 个种子 Founder 入驻，提供 "Listed on Loka" 认证徽章 + 免费 AI 分析报告作为激励。

#### 阶段二：增长期——内容飞轮 + 社交裂变

**1) 报告分享机制（产品驱动增长）**
- Super Agent 生成的 PDF 报告自带 Loka 品牌水印和平台链接。
- 用户分享报告到社交媒体 → 读者被报告质量吸引 → 点击链接注册使用 → 新用户生成自己的报告 → 继续分享。形成"内容即获客"的自然循环。

**2) 社交拉新**
- Chats 的群聊天然是拉新工具：用户想加朋友一起讨论，就得邀请对方注册。
- 信用分系统激励：邀请有效新用户 +30 分，500 分解锁高级功能，正向循环。

**3) SEO + 长内容沉淀**
- 将 AI 生成的高质量分析报告做成可被搜索引擎索引的公开页面。
- 用户搜索"XX 公司分析"或"XX 预测市场"时，Loka 的报告出现在搜索结果中，自然引流。

#### 阶段三：扩展期——从投资到通用协作

- 投资分析用户积累到一定量级后，逐步推出竞品调研、行业研究、团队任务协作等场景。
- 引入团队版（Team Plan），支持企业内多人 + Agent 协作工作空间。
- 建立 Agent 市场（Agent Marketplace），允许用户创建和分享自定义 Agent。

### 1.6 商业化与消耗模型

#### 1. 核心商业模式

平台采用"**基础订阅 + 积分消耗**"的混合商业模式。

- **单一结算单位：** 平台统一使用 **Credit（积分）** 作为衡量所有 AI 算力消耗和付费结算的唯一标的。
- 订阅制为用户提供每个月的固定 Credit 额度，超出部分可直接通过充值购买 Credit 来补充。

#### 2. 积分消耗规则

根据任务复杂度和底层 LLM Token 实际使用量进行动态计算：

- **轻度交互** (Chats 基础对话)：例如单条普通文本回复，消耗少量 Credit（如 1-3 Credit）。
- **中度任务** (单 Agent 资讯检索)：例如查询某公司最新新闻、拉取单一图表，消耗中等 Credit（如 10-20 Credit）。
- **重度任务** (Super Agent 多轮共识投研)：涉及调度多个 Agent 查阅海量外网数据、交叉验证并生成长文报告。根据实际处理的上下文长度，单次深度分析动态消耗大量 Credit（如 100-500 Credit）。

#### 3. 会员阶梯与权益设计

| 会员等级 | 目标人群 | 费用 | Credit 额度与核心权益 |
|----------|----------|------|----------------------|
| **Free (免费版)** | 体验用户、轻度参与者 | $0 / 月 | • 注册即赠送一次性 Credit 额度（如 20,000 Credit）<br>• 额度耗尽后，每日仅恢复极少量额度（仅维持基础群聊） |
| **Pro (专业版)** | 高频交易员、研究员 | $20 / 月 | • 每月自动刷新大额专属 Credit 额度（如 150,000 Credit）<br>• 解锁最高级推理模型（如 deepseek-v3）<br>• 享有优先的算力调度队列 |
| **Team (团队版)** | 初创团队、投资机构 | 阶梯定价 | • 共享团队级别的巨量 Credit 资源池<br>• 支持私有化数据接入 |

#### 4. 额度充值

对于重度使用 Super Agent 的 Pro 用户，如果当月额度提前耗尽，平台支持随时进行额外的算力充值。

**充值路径：** 用户使用法币（信用卡/Stripe）或稳定币购买额度。

## 3. Super Agent

### 3.1 机制

Super Agent 会根据意图调动多个专业 Agent 并行工作，交付结构化结果。即：

```
用户输入任务 -> SuperAgent 判断需要调用的能力 -> 调用相应 Agent 能力 -> 组合答案，输出结果
```

### 3.2 Agent 路由

| 类型 | 任务 | 调用类型 | 输出 |
|------|------|----------|------|
| **通用类** | 简单通用对话，例如"hi" | 单agent的通用能力 | 通用回复 |
| **搜索类** | 搜索市场情绪，例如"了解市场对某个股票的看法" | Signal Radar | |
| | 搜索资产概况信息，例如"某只股票今天情况" | Daily Agent | |
| **分析类** | 深度分析某个资产背景信息、项目情况 | 投资分析 Agent | |
| | 某个资产投资建议 | 投资分析 Agent | |
| **模拟预测类** | 模拟巴菲特投资、给出投资建议 | AI Hedge Fund | |
| | 模拟预测市场 | | |
| **交易类（待定）** | 交易股票、crypto、polymarket资产 | | |

### 3.3 协作模式

- **Auto:** 自动路由，选择适合的模式
- **Fast:** 常规路由调用对应 Agent 能力
- **Roundtable:** 在调用 Agent 能力后，还要进行多 Agent 共识

### 3.4 资产类别

- 股票类
- Crypto 资产
- Polymarket

### 3.5 输出

#### a. 思考流

思考流为一个通用的模版，根据调用的 Agent 类型展示对应的模块，未调用就不显示。

**思考流 = 【搜索】+【分析】+【模拟】+【共识】+【完成】**

| 类型 | 调用 Agent | 思考流 |

## 4. 社交网络 (Community)

### 4.1 定位

Chats 是 Loka 的第二卖点。核心差异是 AI Agent 作为群组一等成员，参考 Teamily 的"人机共存"模式。

**用户为什么用 Loka Community 而不用 Telegram / 微信 / Slack：**

| 能力 | Loka Community | 传统 IM（即使接了 Bot） |
|------|---------------|----------------------|
| AI Agent 作为群成员 | ✅ 有名字有角色，可被 @，主动参与讨论 | ❌ Bot 是附属工具，被动应答 |
| 群内执行任务 | ✅ @Agent 直接出分析报告、做调研、整理纪要 | ❌ 不支持 |
| 自动执行 | ✅ Agent 自动完成任务并输出结果，自动 @ 相关人员通知，不需要人工盯着 | ❌ Bot 需要人手动触发，无法主动协调流程 |
| 原生富交互 | ✅ 分析报告、数据卡片、图表直接在对话流中渲染，点击即可展开/导出 | ❌ Telegram Bot 只能返回纯文本或简单 Markdown，卡片和交互能力受限 |
| 跨群记忆 | ✅ Agent 记得之前在别的群讨论过什么 | ❌ 无状态 |
| 分析报告联动 | ✅ Super Agent 的报告一键转入群讨论 | ❌ 需要手动截图转发 |

### 4.2 群类型

- **项目/主题群：** 围绕某个项目或话题（如"Q2 投资策略""竞品调研"）创建，可邀请 AI Agent 参与
- **团队群：** 同事之间的工作协作群，Agent 辅助推进任务
- **自由群：** 用户自建社交群
- **私信：** 1:1 对话（人对人）

### 4.3 Agent 职能分类

群内 Agent 按职能分为以下类别：

| 类别 | 能力 | 群内用法示例 |
|------|------|-------------|
| 数据分析 | 读取市场/财务数据，做量化分析、估值评估 | "帮我看一下这个公司的收入趋势" |
| 项目分析 | 调研公司/项目基本面、团队背景、商业模式 | "这个项目的创始团队背景怎么样" |
| 产品分析 | 竞品对比、产品功能拆解、用户反馈汇总 | "帮我对比一下这两个竞品" |
| 风险评估 | 识别风险因素、给出风险评分、提出预警 | "这个投资有什么潜在风险" |
| 新闻搜集 | 追踪实时新闻、社交媒体舆情、行业动态 | "最近有什么相关的新闻" |

Agent 可被同时 @ 并行工作（Social Brain 机制），工作中显示分步进度。Agent 记忆跨会话持久化。

## 5. Market 与公司入驻

### 5.1 Market（项目展示）

Market 展示一些初创项目供用户浏览。数据来源：

- **TrustMR API 同步：** 以 Loka 品牌展示，不暴露数据源。
- **Founder 主动入驻：** 可获得 "Listed on Loka" 徽章。

每个项目支持调起 Super Agent 做分析，或直接联系 Founder。

Discover 是平台的一个功能模块。平台的核心价值在于 Super Agent 的通用分析能力和 Chats 的协作体验。

### 5.2 公司入驻

Founder 可以将公司上架到 Loka，获得曝光。

**入驻信息：**
- **必填：** 公司名称、简介、行业、阶段、官网、创始人 Twitter 或 LinkedIn
- **选填：** Pitch、团队介绍、产品链接、是否开放联系方式

**审核流程：** AI 预审 → 人工复核（1-3 工作日）→ 上架展示。

**Founder 联系路径：**
- 平台内私信（自动创建 Chats 对话）
- 外部联系方式（Founder 可选择公开，可限制仅高信用分用户可见）
