## 1. 产品概览

| 模块 | **Loka Asset Launchpad** |
| --- | --- |
| **核心隐喻** | **"Web3 版的 Kickstarter / AngelList"** |
| **设计目标** | 让用户像参与众筹一样，通过购买打折的现金流票据，资助 AI 项目方扩大生产。 |
| **视觉重心** | **融资进度 (Progress)** + **项目叙事 (Story)** + **财务透明 (Data)** |

### 1.1 核心角色

| 角色 | 说明 |
| --- | --- |
| **投资者 (Investor)** | 连接钱包即可投资，无最低 / 最高单笔限额（v1 不设限）。通过信用分获得分级权益。 |
| **项目方 (Issuer)** | 需完成企业验证并获得 SBT 后才可发起募资。信用分驱动抵押率与平台费。 |
| **平台 (Loka)** | 提供基础设施：SPV 隔离、智能合约托管、AI 风控、信用体系、治理框架。 |

### 1.2 技术栈

- **前端：** React 19 + TypeScript + Vite + Tailwind CSS + Recharts
- **后端：** Express 5 + TypeScript + Prisma ORM + SQLite（开发期） + JWT + Socket.IO + Zod
- **AI 服务：** lingyaai.cn (OpenAI 兼容) + deepseek-v3 + SSE 流
- **链上（规划）：** 智能合约资金池 / SBT / RWA NFT

## 5. 募资机制

### 5.1 项目发起准备

| 参数 | 约束 | 说明 |
| --- | --- | --- |
| 目标金额 (Hard Cap) | $10,000 - $1,000,000 | 达到后立刻关闭认购 |
| 最低金额 (Soft Cap) | ≥ 目标金额的 50% | 未达到则募资失败 |
| 募资周期 | 7 - 90 天 | 项目方自行设定 |
| 承诺收益 (APY) | 项目方自定 | 需在申请时明确 |
| 还款周期 | 1 - 12 个月，按月还款 | 项目方自定 |
| 分红规则 | 提前设定分润金额 + 分润周期 | 例如每月返 $1,100 |

### 5.2 募资进行中

- **投资款流向：** 打入平台安全合约池
- **灵活撤资：** 募资成功（敲定）前，用户可随时增投或撤回，**无撤资惩罚**
- **早鸟优惠（可选）：** 前 24 小时认购 或 高信用分用户，额外 0.1% 折扣

### 5.3 募资结果判定

| 场景 | 条件 | 处理 |
| --- | --- | --- |
| **❌ 失败** | 周期结束时未达 Soft Cap | 合约自动退款，本金无损原路退回 |
| **✅ 成功 (达硬顶)** | 周期内任意时刻达到 Hard Cap | 立刻关闭认购，进入锁定期 |
| **✅ 成功 (达软顶)** | 周期结束时达到 Soft Cap 但未达 Hard Cap | 宣告成功，进入锁定期 |

### 5.4 募资成功 → 锁定期

- **状态固化：** 资金池封盘上锁
- **禁止操作：** 不可追加投入，不可提前提取本金
- **投资者权益：** 静默跟进资产状态，等待项目方按约定周期打入分红与本金回款

### 5.5 提前还款规则

项目方可以提前还清全部债务。当发生提前还款时：

- **投资者获得全部预期收益**（按原始约定的 APY x 完整期限计算），而非按实际持有天数折算
- 项目方信用分 +40（提前还清 bonus）+ +25（提前 ≥ 7 天还款 bonus）
- 示例：约定 12 个月 15% APY，项目方第 6 个月还清，投资者仍获得完整 15% 收益

## 6. 项目方申请流程 (Apply Flow)

### 6.1 入口

- 市场页面增加 **`Apply`** 按钮
- 点击后跳转至 Groups 模块，自动创建一个**专属申请群组 (Apply Group)**

### 6.2 Apply Group 设计

Apply Group 与普通社交 Group 不同，是**任务驱动型对话群组**：

| 属性 | 普通 Group | Apply Group |
| --- | --- | --- |
| 类型标记 | `social` | `apply` |
| 成员 | 用户 + 其他用户 | 用户 + 2 个 AI Agent |
| 对话目的 | 自由讨论 | 引导完成企业验证 / 项目申请 |
| 生命周期 | 长期 | 完成后归档 |
| 进度追踪 | 无 | 有步骤进度条 |

### 6.3 群组内 AI Agent 组成

> 共 2 个 Agent，不设人工审核节点。

| Agent | 角色 | 职责 |
| --- | --- | --- |
| **Guide Agent（引导员）** | 主要交互方 | 通过自然语言对话引导用户逐步完成信息填报、文件上传、账户授权。负责解答疑问、校验格式、推进流程。 |
| **Reviewer Agent（审核员）** | 自动审核方 | 在每个关键节点自动介入，对提交的材料进行合规校验（如执照 OCR、地址格式、收入数据一致性）。输出通过 / 驳回 / 补充材料的决定。 |

### 6.4 申请流程总览

```
用户点击 Apply → 创建 Apply Group → Guide Agent 欢迎 & 判断状态
    ├─ 无 SBT → 进入「企业验证」流程（§7）
    │       完成后 → Mint SBT → Guide Agent 提示可进入项目申请
    └─ 已有 SBT → 直接进入「项目申请」流程（§8）
```

**关键规则：**
- 企业验证只需完成 **一次**，SBT 永久有效且可复用
- 后续发起多次融资无需重新验证企业
- Stripe / PayPal 等收入账户连接属于数据授权行为，**不自动判定通过**，仍需 Reviewer Agent 根据实际流水数据评估

## 7. 企业验证 (Enterprise Verification)

> 流程摘要：上传执照 → 查老板 (UBO Check) → 拿凭证 (Mint SBT)

### 7.1 Step 1: 法律主体认证（文件上传）

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 注册名称 | 企业全称 | ✅ |
| 注册国家/地区 | 下拉选择 | ✅ |
| 注册号 | 公司注册编号 | ✅ |
| 注册地址 | 详细地址 | ✅ |
| 营业执照 | 上传扫描件 | ✅ |
| 地址证明 | 近 3 个月水电单 / 银行对账单 | ✅ |

### 7.2 Step 2: 实际控制人核查 (UBO Check)

- **股权结构披露：** 声明持有 25% 以上股份的所有股东身份
- **KYC：** 核心股东及法人代表上传护照 / 身份证，并进行人脸识别
- 由 Reviewer Agent 自动校验文件完整性与格式

### 7.3 Step 3: 审核与上链

- **第三方合作机构** 进行最终复核（异步，Agent 通知结果）
- 通过后，铸造 **Verified Issuer SBT**（灵魂绑定代币），绑定到项目方钱包
- **拥有 SBT 的钱包才有权限：** 创建资产池、发起募资、提取资金

### 7.4 相关信用分

完成企业验证全流程可获得 **+100 信用分**（详见 §9.1.2）：

| 步骤 | 得分 |
| --- | --- |
| 企业信息填报 | +25 |
| 核心证照上传 | +25 |
| 股权结构披露 | +25 |
| 实际控制人 KYC | +25 |

## 10. 清算机制

### 10.1 立即清算

> 适用场景：恶意跑路、虚假数据、彻底破产等严重违约

**触发条件（满足任一）：**
- 断开数据连接 / API
- 被证实提交虚假财务数据或欺诈
- 到期未还款，且拒绝沟通或债务重组失败

**执行逻辑：**
1. 清算抵押物并折现
2. 按投资比例分配给投资者
3. 项目方信用分 -150（逾期 >30 天）或 -300（虚假数据），冻结 / 封禁账号

### 10.2 待清算（协商重组）

> 适用场景：确实在经营且有流水，但因客户付款延迟等原因暂时还不上

**触发条件：**
- 到期未还款达到指定额度

**处理流程：**
1. 项目方申请延迟收款
2. 平台发起**投资者治理投票**，投票选项：
   - **方案 A：** 降低每月还款额度，延长总周期
   - **方案 B：** 每月收入 100% 还给投资者，直到还清
   - **方案 C：** 追加抵押品
   - **方案 D：** 立即清算
3. 按投票权重（见 §11.3）计算结果，执行最高票方案

## 11. 治理系统

> **核心原则：** 投资者主导治理，项目方不参与自身项目投票。

### 11.1 治理等级

| 信用分 | 等级 | 权限 |
| --- | --- | --- |
| 100-149 | 观察员 | 浏览提案、社区讨论，无投票权 |
| 150-499 | 参与者 | 基础类投票（平台规则、通用事项） |
| 500-999 | 治理者 | 所有投票 + 评论提案 |
| 1000+ | 提案者 | 发起新提案 + 所有投票 |

> 低门槛设计：首次投资 (+30) + 早鸟认购 (+20) = 150 分，即可获得投票权。

### 11.2 治理事项矩阵

| 事项 | 观察员 | 参与者 | 治理者 | 提案者 | 特殊规则 |
| --- | --- | --- | --- | --- | --- |
| 平台规则变更 | 👁️ | ✅ | ✅ | 可发起 | 全体投票 |
| 新项目白名单 | 👁️ | ✅ | ✅ | 可发起 | 全体投票 |
| 待清算项目处理 | 👁️ | ❌ | ✅ | 可发起 | 仅投资者 |
| 立即清算触发 | 👁️ | ❌ | ✅ | 可发起 | 仅投资者 |
| 争议仲裁 | 👁️ | ❌ | ✅ | 可发起 | 利益相关方权重 x2 |
| 平台发展方向 | 👁️ | ✅ | ✅ | 可发起 | 全体投票 |
| 紧急暂停 | 👁️ | ❌ | ✅ | 可发起 | 快速通道 24h |

### 11.3 投票权重

**公式：权重 = 信用分 x 角色系数 x 利益系数**

| 系数 | 情况 | 数值 |
| --- | --- | --- |
| **角色系数** | 投资者 | 1.0 |
| | 项目方（已认证） | 0.8 |
| | 项目方（蓝筹） | 1.0 |
| **利益系数** | 无直接利益 | 1.0 |
| | 持有该项目份额 | 1.5 |
| | 是该项目的项目方 | **0（禁止投票）** |
| | 被仲裁的项目方 | **0（禁止投票）** |

### 11.4 治理奖励

| 行为 | 得分 | 月上限 |
| --- | --- | --- |
| 参与投票 | +5 | 25 分 |
| 投票与最终结果一致 | +10 | 30 分 |
| 评论被点赞 TOP3 | +15 | 15 分 |
| 发起提案被通过 | +30 | 30 分 |
| 参与紧急治理 | +20 | 20 分 |
